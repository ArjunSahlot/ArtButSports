# GCP VM Backend Setup

This guide is for running only the ArtButSports backend on a Google Cloud VM while the frontend runs on Vercel at:

```text
https://artbutsports.vercel.app
```

The VM does not need the Next.js frontend build. It needs the Python backend, the feature table, the CMA metadata CSV, the downloaded corpus images, and a small copy of demo assets for demo queries.

## What Lives Where

The backend reads these files at runtime:

| Purpose | Default path on VM | Required? | How it is made |
| --- | --- | --- | --- |
| Metadata CSV | `data/img-metadata-CC0-clean.csv` | Yes | You already copied this to `data/` |
| Feature table | `data/features/artbutsports_features.npz` | Yes | Build locally with `scripts/build_features.py`, then `scp` to VM |
| Corpus images | `data/vm_images/images/*.jpg` | Yes for local image serving | Download on VM with `scripts/setup_vm_images.py` |
| Image manifest | `data/vm_images/manifest.json` | Yes for local image serving | Generated by `scripts/setup_vm_images.py` |
| Demo query images | `data/demos/*.png` | Yes if demo gallery should query backend | Copy from `frontend/public/demos/` |
| Visualization samples | `data/visualize_samples/*.png` | Optional legacy backend sample route | Copy from `frontend/public/visualize-samples/` if you use backend sample endpoints |

The feature table is the small artifact that contains precomputed vectors. The corpus images are separate and are downloaded on the VM because they are several GB.

## 1. VM Prerequisites

On the VM:

```bash
sudo apt-get update
sudo apt-get install -y git curl ca-certificates build-essential tmux
```

Install `uv`:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source "$HOME/.local/bin/env"
uv --version
uv python install 3.13
```

If the VM is small, add swap before installing/running ML dependencies:

```bash
sudo fallocate -l 8G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h
```

## 3. Create The VM `.env`

Create `~/ArtButSports/.env` on the VM:

```bash
cat > .env <<'EOF'
GEMINI_API_KEY=replace_me
FRONTEND_ORIGIN=https://artbutsports.vercel.app

FEATURE_TABLE_PATH=data/features/artbutsports_features.npz
IMAGE_MANIFEST_PATH=data/vm_images/manifest.json
DEMO_IMAGE_DIR=data/demos
VISUALIZE_SAMPLE_DIR=data/visualize_samples

HOST=0.0.0.0
PORT=8000
EOF
chmod 600 .env
```

Notes:

- `GEMINI_API_KEY` is required at query time because the backend embeds each uploaded/demo query image.
- `FRONTEND_ORIGIN` must match the Vercel production origin exactly for CORS.
- Keep `.env` uncommitted.

## 4. Copy The Feature Table To The VM

The feature table should be built on your faster local machine, not on the VM:

```bash
python scripts/build_features.py --output data/features/artbutsports_features.npz
```

Then copy it to the VM:

```bash
ssh YOUR_VM 'mkdir -p ~/ArtButSports/data/features'
scp data/features/artbutsports_features.npz YOUR_VM:~/ArtButSports/data/features/artbutsports_features.npz
```

Quick sanity check on the VM:

```bash
cd ~/ArtButSports
uv run python - <<'PY'
import numpy as np
p = "data/features/artbutsports_features.npz"
d = np.load(p, allow_pickle=True)
print("rows:", len(d["ids"]))
print("keys:", sorted(d.files))
print("incomplete_summary:", d["incomplete_summary"].item() if "incomplete_summary" in d else None)
PY
```

You should see roughly the number of complete rows from your local feature build.

## 5. Copy Demo Assets Needed By Backend Demo Queries

The Vercel frontend displays demos from `frontend/public/demos`, but when a user clicks one it calls the backend `/query/demo` endpoint with the filename. The backend therefore also needs those same demo files locally.

On the VM:

```bash
cd ~/ArtButSports
mkdir -p data/demos data/visualize_samples
cp frontend/public/demos/* data/demos/
cp frontend/public/visualize-samples/* data/visualize_samples/ 2>/dev/null || true
```

If you do not copy `data/demos`, manual uploads will still work, but demo clicks from the landing gallery will fail.

## 6. Download Corpus Images On The VM

You already copied:

```text
data/img-metadata-CC0-clean.csv
```

Now download the CMA `image_web` images and build the backend image manifest:

```bash
cd ~/ArtButSports
uv run python scripts/setup_vm_images.py \
  --metadata data/img-metadata-CC0-clean.csv \
  --image-dir data/vm_images/images \
  --manifest data/vm_images/manifest.json
```

This is resumable at the file level: rerunning skips images that already exist and have nonzero size. It should finish with a JSON report containing `count`, `missing_count`, and `total_size_gb`.

Run it inside `tmux` so a dropped SSH session does not kill the download:

```bash
tmux new -s artbutsports-images
cd ~/ArtButSports
uv run python scripts/setup_vm_images.py --metadata data/img-metadata-CC0-clean.csv
```

Detach with `Ctrl-b`, then `d`. Reattach with:

```bash
tmux attach -t artbutsports-images
```

Sanity check:

```bash
cd ~/ArtButSports
python3 - <<'PY'
import json
from pathlib import Path
m = json.loads(Path("data/vm_images/manifest.json").read_text())
print("manifest rows:", len(m))
sample = next(iter(m.values()))
print("sample exists:", Path(sample).exists(), sample)
PY
du -sh data/vm_images
```

## 7. Run The Backend Manually First

Start it:

```bash
cd ~/ArtButSports
source .env
scripts/run_backend.sh
```

In another SSH session:

```bash
curl http://127.0.0.1:8000/health
```

Expected shape:

```json
{"ok":true,"features_loaded":true,"images_mapped":24000}
```

The exact `images_mapped` count can be lower if some image downloads failed.

Test an image route:

```bash
ART_ID="$(uv run python - <<'PY'
import json
m = json.load(open("data/vm_images/manifest.json"))
print(next(iter(m.keys())))
PY
)"
curl -I "http://127.0.0.1:8000/images/$ART_ID"
```

Test CORS preflight from Vercel:

```bash
curl -i -X OPTIONS http://127.0.0.1:8000/query \
  -H 'Origin: https://artbutsports.vercel.app' \
  -H 'Access-Control-Request-Method: POST'
```

You should see:

```text
access-control-allow-origin: https://artbutsports.vercel.app
```

## 8. Run As A Systemd Service

Create a service file:

```bash
sudo tee /etc/systemd/system/artbutsports-backend.service >/dev/null <<EOF
[Unit]
Description=ArtButSports FastAPI backend
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=$USER
WorkingDirectory=/home/$USER/ArtButSports
EnvironmentFile=/home/$USER/ArtButSports/.env
Environment=PATH=/home/$USER/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
ExecStart=/home/$USER/.local/bin/uv run fastapi run backend/api/app.py --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

Enable and start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now artbutsports-backend
sudo systemctl status artbutsports-backend --no-pager
```

Logs:

```bash
journalctl -u artbutsports-backend -f
```

Health:

```bash
curl http://127.0.0.1:8000/health
```

## 9. Expose The Backend Publicly With HTTPS

The Vercel site is HTTPS, so the browser will block frontend calls to a plain `http://YOUR_VM_EXTERNAL_IP:8000` backend as mixed content. Use raw `http://127.0.0.1:8000` only for SSH/manual checks. For the live Vercel app, put HTTPS in front of the backend.

In GCP, reserve or keep a stable external IP for the VM, then point a DNS name such as `api.artbutsports.com` at that IP. Open ports `80` and `443` in the VM firewall. You do not need to expose port `8000` publicly if Caddy runs on the same VM.

Install Caddy:

```bash
sudo apt-get install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt-get update
sudo apt-get install -y caddy
```

If you have a domain like `api.artbutsports.com` pointed at the VM:

```bash
sudo tee /etc/caddy/Caddyfile >/dev/null <<'EOF'
api.artbutsports.com {
  reverse_proxy 127.0.0.1:8000
}
EOF
sudo systemctl reload caddy
```

Then set the Vercel frontend environment variable:

```text
NEXT_PUBLIC_API_BASE_URL=https://api.artbutsports.com
```

After changing Vercel env vars, redeploy the Vercel project.

## 10. Final End-To-End Checks

From your laptop:

```bash
curl https://api.artbutsports.com/health
```

or, if you temporarily exposed port `8000` for a manual smoke test:

```bash
curl http://YOUR_VM_EXTERNAL_IP:8000/health
```

In the browser:

1. Open `https://artbutsports.vercel.app`.
2. Open DevTools Network tab.
3. Pick a demo image.
4. Confirm the frontend calls `NEXT_PUBLIC_API_BASE_URL/query/demo`.
5. Upload a fresh image.
6. Confirm the frontend calls `NEXT_PUBLIC_API_BASE_URL/query`.
7. Open Visualize and try each modal; those should call `NEXT_PUBLIC_API_BASE_URL/visualize/step`.

```bash
git pull

journalctl -u artbutsports-backend -n 80 --no-pager

```

For a new feature table:


```bash

```

## Troubleshooting

### `/health` says `features_loaded: false`

The feature table is missing or the path is wrong.

```bash
ls -lh data/features/artbutsports_features.npz
grep FEATURE_TABLE_PATH .env
journalctl -u artbutsports-backend -n 100 --no-pager
```

### `/health` says `images_mapped: 0`

The manifest is missing or the image setup script has not completed.

```bash
ls -lh data/vm_images/manifest.json
uv run python scripts/setup_vm_images.py --metadata data/img-metadata-CC0-clean.csv
sudo systemctl restart artbutsports-backend
```

### Demo clicks fail but uploads work

Copy demo files:

```bash
mkdir -p data/demos
cp frontend/public/demos/* data/demos/
sudo systemctl restart artbutsports-backend
```

### Browser shows CORS errors

Check `.env`:

```bash
grep FRONTEND_ORIGIN .env
```

It should be exactly:

```text
FRONTEND_ORIGIN=https://artbutsports.vercel.app
```

Restart after changing it:

```bash
sudo systemctl restart artbutsports-backend
```

### Query returns `Embedding failed`

The Gemini key is missing or invalid on the VM.

```bash
grep GEMINI_API_KEY .env
journalctl -u artbutsports-backend -n 100 --no-pager
```

### VM runs out of memory during install/startup

Add swap, use a larger VM, or both. The backend loads the feature table and ML models into memory, so a tiny VM can be unstable.
